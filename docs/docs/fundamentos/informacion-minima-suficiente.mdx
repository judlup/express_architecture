# Información Mínima Suficiente

La Información Mínima Suficiente (IMS) es el conjunto de variables y atributos necesarios para comprender una situación y habilitar una decisión específica, sin intentar documentar todos los detalles posibles.

Por qué usar IMS:

- Evita sobrecarga informacional.
- Permite avanzar rápidamente y generar artefactos útiles.
- Facilita la toma de decisiones con evidencia suficiente.

Ejemplos de variables (IMS típicas):

- `proceso` — nombre y alcance del proceso analizado.
- `responsable` — persona o rol con responsabilidad sobre el proceso.
- `área` — unidad organizacional responsable.
- `sistema` — sistemas clave involucrados.
- `herramienta` — herramientas utilizadas (si aplica).
- `métrica` — indicador asociado (si existe) y frecuencia.
- `tiempo` — tiempos relevantes (SLAs, frecuencia de ejecución).
- `entrada` / `salida` — insumos y resultados del proceso.
- `dependencia` — relaciones críticas con otros procesos o sistemas.
- `restricción` — limitantes técnicas, regulatorias o de negocio.
- `decisión asociada` — la decisión que se busca apoyar.

Criterios para considerar que la información es suficiente:

- Responde directamente a la pregunta o decisión objetivo.
- Permite identificar responsables y acciones posibles.
- Reduce la incertidumbre relevante para la decisión.
- Es verificable por actores clave (stakeholders).
- Permite derivar al menos un artefacto visual que comunique el problema.

Cómo aplicar IMS en la práctica:

1. Definir la decisión a tomar (o la hipótesis a validar).
2. Listar las variables mínimas necesarias para esa decisión.
3. Priorizar entrevistas y fuentes para conseguir esas variables.
4. Registrar y validar con stakeholders.
5. Generar un artefacto visual y evaluar si permite la decisión; si no, iterar.

# Guía de Implementación

Esta sección contiene pasos prácticos y recursos para aplicar Express Architecture en tu organización.

Páginas principales:

## Resumen rápido

Esta guía presenta un flujo práctico y las recomendaciones mínimas para aplicar Express Architecture. Si una sección no requiere mayor profundidad, está resumida aquí para facilitar lectura rápida.

---

## Antes de empezar

Antes de aplicar Express Architecture conviene definir alcance, roles e insumos mínimos.

Cuándo usarla:

- Cuando se necesita apoyar una decisión concreta (no documentar todo).
- Cuando hay información dispersa o conocimiento tácito.
- Para levantamientos rápidos o iniciativas de modernización.

Requisitos mínimos e insumos:

- Definir la decisión o pregunta de gestión a resolver.
- Identificar actores clave y responsables.
- Acceso a información primaria: entrevistas, documentos, Excel, sistemas.

Roles recomendados:

- Facilitador / arquitecto
- Sponsor / decisor
- Representantes de negocio y tecnología

---

## Flujo general

1. Contexto y decisión: definir claramente qué se quiere comprender o decidir.
2. Captura estructurada: recolectar información relevante mediante entrevistas, talleres y análisis.
3. Filtrado IMS: aplicar el criterio de Información Mínima Suficiente.
4. Diseño estructural: elegir el artefacto visual adecuado.
5. Levantamiento y construcción: mapear variables y construir el artefacto.
6. Generación visual: producir la representación validable.
7. Validación y victorias tempranas: revisar con stakeholders y documentar mejoras.

---

## Captura estructurada

Métodos recomendados: entrevistas semiestructuradas, talleres colaborativos, observación participante y revisión documental. Priorizar fuentes que aporten las variables IMS definidas para la decisión.

---

## Selección del artefacto

Usar una matriz de selección simple:

- Si se quiere entender un flujo → `Swimlane / BPMN simplificado`.
- Si se quieren ver dependencias → `Mapa de dependencias`.
- Si se quiere medir capacidad → `Dashboard operativo`.
- Si se quieren analizar sistemas → `Matriz proceso-sistema`.
- Si se quiere priorizar → `Matriz de priorización`.

---

## Construcción del artefacto

Buenas prácticas:

- Mantener la visualización simple y enfocada en la decisión.
- Indicar responsables y fuentes de información.
- Vincular el artefacto con otras piezas (matrices, dashboards, mini ADRs).

---

## Validación

Criterios: claridad, utilidad, trazabilidad, facilidad de uso y soporte a decisiones. Validar con un conjunto reducido de stakeholders y ajustar.

---

## Checklist

- Contexto definido
- Decisión identificada
- Actores mapeados
- Procesos levantados
- Sistemas identificados
- Métricas registradas
- Dependencias conectadas
- Artefacto seleccionado y validado
- Victoria temprana registrada

# Antes de empezar

Antes de aplicar Express Architecture conviene definir alcance, roles e insumos mínimos.

Cuándo usarla:

- Cuando se necesita apoyar una decisión concreta (no documentar todo).
- Cuando hay información dispersa o conocimiento tácito.
- Para levantamientos rápidos o iniciativas de modernización.

Cuándo no usarla:

- Cuando se requiere una gobernanza formal exhaustiva desde el inicio.
- En casos donde la única salida válida es una auditoría detallada con esfuerzo prolongado.

Requisitos mínimos e insumos:

- Definir la decisión o pregunta de gestión a resolver.
- Identificar actores clave y responsables.
- Acceso a información primaria: entrevistas, documentos, Excel, sistemas.
- Herramientas para capturar información (plantillas, pizarras, herramientas colaborativas).

Roles recomendados:

- Facilitador / arquiteto: guía el levantamiento y diseño del artefacto.
- Sponsor / decisor: valida objetivos y decisiones.
- Representantes de negocio: aportan contexto y requisitos.
- Representantes de tecnología: identifican sistemas y restricciones.

Nivel de madurez recomendado:

- No se requiere madurez alta; la metodología es aplicable en organizaciones con poca documentación siempre que exista acceso a interlocutores clave.

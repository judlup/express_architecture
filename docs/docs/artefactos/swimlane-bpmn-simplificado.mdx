# Swimlane / BPMN simplificado

## Propósito

Visualizar flujos entre áreas y actores para identificar transferencias de responsabilidad y puntos de fricción.

## Cuándo usarlo

- Para analizar procesos transversales que atraviesan varias áreas.
- Para clarificar responsables en handoffs y transferencias.

## Qué captura

- Actividades y tareas principales.
- Responsables por carril.
- Entradas y salidas por actividad.
- Puntos de transferencia y tiempos estimados.

## Cómo construirlo

1. Definir el alcance del flujo.
2. Asignar carriles a áreas o roles.
3. Mapear actividades y transferencias.
4. Validar con participantes.

## Errores comunes

- Incluir demasiado detalle operativo en la primera versión.
- No indicar claramente responsables en cada actividad.

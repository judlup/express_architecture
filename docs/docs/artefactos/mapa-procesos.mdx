# Mapa de Procesos

## Propósito

Representar procesos y subprocesos para entender el flujo de valor, entradas, salidas y responsables.

## Cuándo usarlo

- En diagnósticos operativos y de mejora continua.
- Para identificar puntos de decisión y flujos críticos.

## Qué captura

- Procesos y subprocesos.
- Actividades clave y responsables.
- Entradas y salidas.
- Puntos de decisión y tiempos asociados.

## Cómo construirlo

1. Mapear el proceso a nivel macro.
2. Descomponer en subprocesos y actividades críticas.
3. Identificar responsables y sistemas asociados.
4. Validar secuencia y tiempos con actores.

## Ejemplo de uso

- Diagnóstico para identificar cuellos de botella y dependencias.

## Errores comunes

- Mapear demasiado detalle en la primera iteración; preferir IMS.
- No validar con los responsables reales.

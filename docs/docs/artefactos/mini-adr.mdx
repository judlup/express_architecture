# Mini ADR

## Propósito

Registrar decisiones técnicas o de diseño de forma ligera y trazable.

## Estructura mínima

- **Contexto:** qué motiva la decisión.
- **Decisión:** enunciado claro.
- **Alternativas consideradas:** breve lista.
- **Consecuencias:** impacto esperado.
- **Responsables:** quién toma la decisión.
- **Fecha:** registro temporal.
- **Evidencias usadas:** referencias o artefactos vinculados.

## Cómo usarlo

- Mantener el documento corto y enlazar artefactos que soporten la decisión.

## Errores comunes

- Documentar decisiones sin responsables claros.
- No vincular evidencia ni alternativas.

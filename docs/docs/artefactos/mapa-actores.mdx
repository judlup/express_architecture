# Mapa de Actores

## Propósito

Representar quién participa, quién decide, quién ejecuta y quién se ve afectado por un proceso o servicio.

## Cuándo usarlo

- Al iniciar levantamientos para entender responsabilidades.
- Para clarificar roles en procesos críticos o proyectos de cambio.

## Qué información captura

- Actores (roles, equipos, personas).
- Tipo de participación (decide, ejecuta, informa, afecta).
- Canales de comunicación y frecuencia.
- Dependencias entre actores (hijos/padres, proveedores).

## Cómo construirlo

1. Listar actores a partir de entrevistas y documentación.
2. Clasificarlos por rol y grado de influencia.
3. Dibujar relaciones (quién informa a quién, quién decide).
4. Validar con stakeholders clave.

## Ejemplo

- Equipo de Operaciones (ejecuta)
- Gerencia (decide)
- Soporte (informa)

## Errores comunes

- Confundir actor con rol técnico (usar la descripción adecuada).
- No incluir actores externos críticos (proveedores, clientes).
- Representar demasiados detalles que dificultan la lectura; priorizar IMS.

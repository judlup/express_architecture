# Matriz Proceso-Sistema

## Propósito

Conectar procesos con los sistemas que los soportan para detectar dependencias, huecos y duplicidades.

## Cuándo usarlo

- Al planear migraciones o integraciones.
- Para priorizar mejoras en sistemas críticos.

## Qué captura

- Proceso y actividad.
- Sistema(s) asociado(s).
- Responsable técnico o de proceso.
- Nivel de soporte (completo/parcial/manual).
- Notas sobre integraciones y restricciones.

## Cómo construirlo

1. Listar procesos priorizados.
2. Identificar sistemas que intervienen en cada proceso.
3. Registrar soporte, responsables y notas.
4. Validar con equipos de tecnología.

## Errores comunes

- Olvidar sistemas no oficiales (hojas de cálculo, herramientas locales).
- No actualizar la matriz tras cambios en sistemas.

# Matriz Proceso-Herramienta

## Propósito

Detectar duplicidad de herramientas, puntos de dependencia y oportunidades de consolidación.

## Cuándo usarlo

- Revisión de portafolio de herramientas.
- Preparación para licenciamiento o compras.

## Qué captura

- Proceso y actividad.
- Herramienta(s) usada(s).
- Propietario / administrador.
- Nivel de criticidad / SLA.
- Alternativas y notas sobre integración.

## Cómo construirlo

1. Inventariar herramientas conocidas y procesos que soportan.
2. Identificar propietarios y dependencias.
3. Clasificar por criticidad y duplicidad.
4. Generar recomendaciones de consolidación.

## Errores comunes

- Omitir herramientas ad-hoc (hojas, macros).
- Evaluar sin datos de uso (preguntar a usuarios).

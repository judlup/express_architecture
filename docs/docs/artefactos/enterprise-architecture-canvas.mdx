# Enterprise Architecture Canvas

## Propósito

Proveer una vista consolidada que conecte procesos, aplicaciones, datos e infraestructura para la toma de decisiones de alto nivel.

## Cuándo usarlo

- Planeación de roadmap tecnológico.
- Evaluación de impacto de cambios a gran escala.

## Qué captura

- Procesos y capacidades.
- Aplicaciones y servicios.
- Datos críticos y flujos.
- Infraestructura y requisitos de seguridad.
- Roadmap y prioridades.

## Cómo construirlo

1. Reunir stakeholders de negocio y tecnología.
2. Mapear elementos en capas (procesos, aplicaciones, datos, infra).
3. Priorizar iniciativas y documentar dependencias.

## Errores comunes

- Intentar profundizar en exceso en una sola iteración.
- No conectar el canvas con decisiones concretas o iniciativas.

# Infrastructure Canvas

## Propósito

Representar componentes de infraestructura, servicios y controles necesarios para desplegar y operar soluciones.

## Cuándo usarlo

- Planes de migración a la nube.
- Evaluación de capacidad e impacto de cambios infraestructurales.

## Qué captura

- Recursos (compute, storage, red).
- Servicios y dependencias externas.
- Controles de seguridad y compliance.
- Responsables operativos.

## Cómo construirlo

1. Inventariar recursos y servicios críticos.
2. Mapear dependencias y pisos de resiliencia.
3. Documentar requisitos de seguridad y responsables.

## Errores comunes

- No considerar dependencias externas (proveedores, APIs).
- No registrar límites operativos y SLAs.

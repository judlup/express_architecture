# Mapa de Dependencias

## Propósito

Hacer visibles relaciones críticas entre áreas, procesos, sistemas y decisiones para gestionar riesgos y continuidad.

## Cuándo usarlo

- Análisis de riesgos y continuidad operativa.
- Evaluación previa a cambios de infraestructura o procesos.

## Qué captura

- Entidades (áreas, procesos, sistemas).
- Relaciones y dirección de la dependencia.
- Nivel de criticidad y propietario.

## Cómo construirlo

1. Identificar entidades críticas.
2. Registrar dependencias y su dirección.
3. Asignar criticidad y responsable.
4. Validar impactos y plan de mitigación.

## Errores comunes

- No representar la dirección de la dependencia (quién depende de quién).
- Subestimar dependencias indirectas.

# Solution Architecture Canvas

## Propósito

Estructurar la propuesta de solución técnica, integraciones, riesgos y decisiones relevantes para su implementación.

## Cuándo usarlo

- Diseño de soluciones para una iniciativa o proyecto específico.
- Evaluación de alternativas técnicas antes de una implementación.

## Qué captura

- Componentes de la solución.
- Integraciones y dependencias.
- Riesgos técnicos y mitigaciones.
- Decisiones y alternativas (mini ADR).

## Cómo construirlo

1. Definir el alcance de la solución.
2. Mapear componentes y dependencias.
3. Identificar riesgos y registrar decisiones clave.

## Errores comunes

- No documentar alternativas consideradas.
- No vincular decisiones con evidencia o responsables.

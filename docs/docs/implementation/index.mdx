# Guía de Implementación (resumen)

Esta guía ofrece pasos prácticos para aplicar Express Architecture utilizando el principio de Información Mínima Suficiente.

## Requisitos mínimos

- Definir la decisión o conjunto de decisiones a soportar.
- Contar con un facilitador y representantes de las áreas clave.
- Acceso básico a información sobre procesos y sistemas.

## Roles involucrados

- Facilitador/arquitecto
- Responsable de proceso
- Representante técnico
- Decisor / Sponsor

## Flujo recomendado

1. Identificar el contexto y la decisión objetivo.
2. Mapear procesos y áreas usando IMS.
3. Identificar sistemas, actores y flujos de información.
4. Construir artefactos visuales mínimos (canvas, diagramas, matriz de decisiones).
5. Detectar brechas, riesgos y oportunidades.
6. Priorizar acciones y documentar acuerdos (mini ADR).
7. Ejecutar, medir y retroalimentar para iterar.

## Checklist de implementación

- ¿La información capturada responde a la decisión objetivo?
- ¿Se identificaron los actores clave?
- ¿Los artefactos permiten trazar la decisión a evidencia?

## Errores comunes

- Documentar en exceso sin definir la decisión.
- No involucrar a decisores relevantes.
- Confundir detalle técnico con información necesaria para la decisión.
